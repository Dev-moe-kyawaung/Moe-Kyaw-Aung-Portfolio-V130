Common Questions

Frequently Asked Questions

What is your primary Android development stack?
My primary stack is Kotlin + Jetpack Compose for UI, ViewModel + StateFlow for state management, Hilt for dependency injection, Room for local persistence, Retrofit + OkHttp for networking, and Firebase for backend services. All built on Clean Architecture with MVVM or MVI patterns.
Do you work remotely or on-site?
I work 100% remotely and have done so successfully for 6+ years across teams in Asia, Europe, and North America. I'm experienced with async communication (Slack, Jira, GitHub), timezone overlap, and distributed team workflows using Agile/Scrum methodologies.
How do you handle app security in Android?
I follow OWASP Mobile Top 10 guidelines: certificate pinning with OkHttp, encrypted SharedPreferences with EncryptedSharedPreferences, biometric authentication, ProGuard/R8 obfuscation, root detection, tamper detection, and secure key storage using Android Keystore system.
Can you set up a CI/CD pipeline for Play Store releases?
Yes — I specialize in GitHub Actions and Azure DevOps pipelines for Android. This includes automated unit/UI tests, lint checks, APK/AAB builds, signing, version bumping, and direct deployment to Play Store internal/alpha/beta/production tracks using Fastlane or Play Publisher API.
What is your availability for new projects?
I'm currently open to new opportunities — both full-time senior positions and freelance/contract Android engagements. I can start within 2 weeks of a signed agreement. Please contact me via email or Telegram to discuss your project timeline.
Do you provide source code ownership and documentation?
Absolutely. All code I write for clients is fully documented (KDoc), includes inline comments for complex logic, and comes with a README covering architecture decisions, setup instructions, and module documentation. Full source code and IP ownership transfers upon project completion.
Can you migrate a Java Android app to Kotlin + Compose?
Yes, I have extensive experience migrating legacy Java Android codebases to modern Kotlin with Jetpack Compose. I use an incremental migration strategy: introduce Kotlin interop first, then migrate feature-by-feature to Compose, while maintaining app stability throughout. This typically results in 30-50% crash reduction and significantly improved developer velocity.
